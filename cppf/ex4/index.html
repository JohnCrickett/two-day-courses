Exercise 4: Composite

A class is not a useful unit of design, while clusters of classes in particular patterns are. This exercise is about using several collaborating classes.

Take a look at these files:

  • directory_entry.hpp (html, raw)
  • file.hpp (html, raw)
  • directory.hpp (html, raw)
  • directory_entry_visitor.hpp (html, raw)
with the corresponding implementation files:
  • directory.cpp (html, raw)
  • file.cpp (html, raw)
and finally, a Make file and a demo file showing an example how this stuff might be used:
  • directory_demo.cpp (html, raw)
  • Makefile (html, raw)
Using these files as a starting point, please update the stuff so each directory entry can also have a date. Feel free to use a date class from exercise 2.

Optional 1

Create a new visitor subclass that can visit the directory structure and create an xml representation.

Optional 2

Create a new visitor subclass that can visit each entry in the directory structure and "touch" the entries so the dates are all the same.
