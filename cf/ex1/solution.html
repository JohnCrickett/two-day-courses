Solution 1: A Brief Tour

Here are some possible solutions to the exercise. But first of all...

The suggested commands for compiling and running depends of course on the compiler that you are using. If you are using gcc then you might want to compile like this: 

gcc -std=c99 -Wall -Wextra -pedantic foo.c
./a.out
Try gcc --help and man gcc for more options. Or if you are using a compiler from Microsoft you might want to consider something like: 
cl /Wall foo.c 
foo.exe
and do cl /? for more options.

1. Hello World!

/* hello.c */

#include <stdio.h>

int main(void) 
{
    puts("Hello World!");
    return 0;
}
To compile and run: 
gcc hello.c
./a.out
Here is a complete code sample: hello.c.

2. Print Arguments

/* printargs.c */

#include <stdio.h>

int main(int argc, char * argv[]) 
{
    for (int at = 0; at != argc; ++at) 
        printf("argv[%d]=%s\n", at, argv[at]);
    return 0;
}
To compile and run: 
gcc -o printargs printargs.c
./printargs foo bar
Here is a complete code sample: printargs.c.

3. Linking together a program

The header file: 
/* sayhello.h */

void sayhello(void);
Notice that this is the proper way of writing a prototype, it is acceptable but not entirely correct to do 'void sayhello();'

The implementation: 

/* sayhello.c */

#include "sayhello.h"
#include <stdio.h>

void sayhello(void) 
{
    puts("Hello");
}
The order of include statements is important. First you should include your own header file, then include header files from your project and then from the platform and standard library. The main reason for this is that it increases that chance of making selfcontained headerfiles.

The main function 

/* main.c */

#include "sayhello.h"

int main(void) 
{
    sayhello();
}
Notice that there is no return statement at the end of the main function. Is that valid code?

To compile and link 

  gcc -c sayhello.c
  gcc -c main.c
  gcc -o main sayhello.o main.o
Here are complete code samples: main.c, sayhello.h and sayhello.c.
